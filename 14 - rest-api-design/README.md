# Online Store REST API

A Node.js / Express REST API that demonstrates REST design principles covered in the Last Mile Program.

---

## Step 1 – Conceptual Setup

### Subtask 1.1 – Resource Identification

The API manages **four primary resources**:

| #   | Resource     | Description                           |
| --- | ------------ | ------------------------------------- |
| 1   | **products** | Items available for purchase          |
| 2   | **orders**   | Purchase transactions placed by users |
| 3   | **users**    | Registered customers                  |
| 4   | **reviews**  | Product reviews written by users      |

### Subtask 1.2 – Base URI & Versioning

```
http://localhost:3000/v1
```

The `/v1` version prefix ensures **backward compatibility** – if breaking changes are needed in the future, a new `/v2` prefix can be introduced while existing clients continue using `/v1` without any changes.

---

## Step 2 – Designing Read Operations (GET)

### Subtask 2.1 – Retrieve Collection

```
GET /v1/products
```

Returns the full list of products.

**Response `200 OK`:**

```json
{
  "total": 5,
  "page": 1,
  "limit": 5,
  "results": [ { "id": 101, "name": "Wireless Headphones", ... } ]
}
```

---

### Subtask 2.2 – Retrieve Single Resource (Path Parameters)

```
GET /v1/products/:id
GET /v1/products/101
```

A **path parameter** (`:id`) uniquely identifies the resource within the URL path.

**Response `200 OK`:**

```json
{
  "id": 101,
  "name": "Wireless Headphones",
  "category": "electronics",
  "price": 79.99,
  "stock": 50
}
```

**Response `404 Not Found`** (when id does not exist):

```json
{ "error": "Product not found." }
```

---

### Subtask 2.3 – Filtering, Sorting & Pagination (Query Parameters)

```
GET /v1/products?category=electronics&sort=price_asc&limit=10
GET /v1/products?category=electronics&sort=price_asc&limit=10&page=2
```

**Query parameters** refine the result set without creating new endpoints:

| Parameter  | Purpose                                       | Example values                        |
| ---------- | --------------------------------------------- | ------------------------------------- |
| `category` | Filter by product category                    | `electronics`, `books`, `footwear`    |
| `sort`     | Order results                                 | `price_asc`, `price_desc`, `name_asc` |
| `limit`    | Maximum number of items per page (pagination) | `10`, `20`                            |
| `page`     | Page number to retrieve                       | `1`, `2`, `3`                         |

**Response `200 OK`:**

```json
{
  "total": 2,
  "page": 1,
  "limit": 10,
  "results": [
    {
      "id": 104,
      "name": "USB-C Hub",
      "category": "electronics",
      "price": 29.99
    },
    {
      "id": 101,
      "name": "Wireless Headphones",
      "category": "electronics",
      "price": 79.99
    }
  ]
}
```

## Step 3 – Designing Write Operations (POST, DELETE)

### Subtask 3.1 – Create Resource (POST)

```
POST /v1/orders
Content-Type: application/json
```

The client submits a **JSON Payload** in the request body – the ID is generated by the server.

**Request body:**

```json
{
  "userId": 1,
  "productId": 101,
  "quantity": 2
}
```

**Response `201 Created`:**

```json
{
  "id": 4,
  "userId": 1,
  "productId": 101,
  "quantity": 2,
  "totalPrice": 159.98,
  "status": "pending"
}
```

**Other status codes:**

- `400 Bad Request` – missing required fields
- `404 Not Found` – product does not exist
- `422 Unprocessable Entity` – invalid quantity

---

### Subtask 3.2 – Remove Resource (DELETE)

```
DELETE /v1/orders/:id
DELETE /v1/orders/2
```

**Response `204 No Content`** – resource deleted, no body returned.  
**Response `404 Not Found`** – order does not exist.

---

## Step 4 – Hierarchy & Nested Resources

### Subtask 4.1 – Parent-Child Relationship

```
GET /v1/users/:userId/orders
GET /v1/users/1/orders
```

The nested URL structure expresses the **parent-child relationship**: orders belong to a user.  
The `:userId` path parameter scopes the query to that specific user.

**Response `200 OK`:**

```json
{
  "userId": 1,
  "userName": "Alice Johnson",
  "total": 2,
  "orders": [
    {
      "id": 1,
      "productId": 101,
      "quantity": 1,
      "totalPrice": 79.99,
      "status": "shipped"
    },
    {
      "id": 3,
      "productId": 104,
      "quantity": 1,
      "totalPrice": 29.99,
      "status": "delivered"
    }
  ]
}
```

---

### Subtask 4.2 – Reviews for a Product (Nested Resource)

```
GET /v1/products/:productId/reviews
GET /v1/products/101/reviews
```

Reviews are **contextually linked** to a product – the nesting makes the relationship explicit in the URL.

**Response `200 OK`:**

```json
{
  "productId": 101,
  "total": 2,
  "reviews": [
    {
      "id": 1,
      "userId": 1,
      "rating": 5,
      "comment": "Excellent sound quality!"
    },
    {
      "id": 2,
      "userId": 2,
      "rating": 4,
      "comment": "Good value for the price."
    }
  ]
}
```

## Step 5 – Final Review & HTTP Status Codes

### Subtask 5.1 – Mapping HTTP Status Codes

| Scenario                                                   | HTTP Status Code              |
| ---------------------------------------------------------- | ----------------------------- |
| Resource fetched successfully                              | **200 OK**                    |
| New resource created successfully                          | **201 Created**               |
| Resource deleted successfully (no body)                    | **204 No Content**            |
| Required fields missing in request body                    | **400 Bad Request**           |
| Authentication required but not provided                   | **401 Unauthorized**          |
| Authenticated but not allowed to access resource           | **403 Forbidden**             |
| Requested resource does not exist                          | **404 Not Found**             |
| Request data is syntactically valid but semantically wrong | **422 Unprocessable Entity**  |
| Unexpected server-side failure                             | **500 Internal Server Error** |

## Project Structure

```
14 - rest-api-design/
├── package.json
└── src/
    ├── server.js           ← Entry point; mounts all /v1 routes
    ├── data/
    │   └── store.js        ← In-memory data store (products, orders, users, reviews)
    └── routes/
        ├── products.js     ← GET /v1/products, GET /v1/products/:id, GET /v1/products/:productId/reviews
        ├── orders.js       ← POST /v1/orders, DELETE /v1/orders/:id
        └── users.js        ← GET /v1/users/:userId/orders
```

## Quick Start

```bash
# Install dependencies
npm install

# Start the server
npm start

# Development mode (auto-restart on file changes)
npm run dev
```

Server runs on **http://localhost:3000**.

---

## Example curl Commands

```bash
# Subtask 2.1 – Get all products
curl http://localhost:3000/v1/products

# Subtask 2.2 – Get a single product
curl http://localhost:3000/v1/products/101

# Subtask 2.3 – Filter, sort & paginate
curl "http://localhost:3000/v1/products?category=electronics&sort=price_asc&limit=10"

# Subtask 3.1 – Create an order
curl -X POST http://localhost:3000/v1/orders \
  -H "Content-Type: application/json" \
  -d '{"userId":1,"productId":102,"quantity":1}'

# Subtask 3.2 – Delete an order
curl -X DELETE http://localhost:3000/v1/orders/2

# Subtask 4.1 – Get all orders for a user
curl http://localhost:3000/v1/users/1/orders

# Subtask 4.2 – Get reviews for a product
curl http://localhost:3000/v1/products/101/reviews
```
